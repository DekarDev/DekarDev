## 🧭 DDS — Dekar Development Standards

DDS is my personal development methodology focused on creating software that is modular, documented, scalable and easy to maintain over time.

The goal is to build projects that can be understood, improved and continued even after long pauses, using clear structures, technical documentation, diagrams, version control and clean development practices.

Core principles:

- Modular code.
- Clear documentation.
- Low coupling between components.
- Safe database operations.
- Meaningful version control.
- Continuous improvement.
- Real solutions for real operational problems.
